# Supervised Fine-Tuning warm-up

In the previous chapter, we saw that before applying Reinforcement Learning, it might be a good idea to start
with a quick Supervised Fine-Tuning warm-up phase, to teach `LiquidAI/LFM2-2.6B` our format and generated
~200 examples using  `openai/gpt-5-mini`.

Now we are going to perform the actual SFT training.

## SFT with PRIME-RL

Verifiers provides a simple RL trainer but it's not thougt for Supervised Fine Tuning.

So, to proceed with training, there are several options we can consider. In this course, we use [PRIME-RL](https://docs.primeintellect.ai/prime-rl), a framework for large-scale asynchronous reinforcement learning that also offers SFT
training. As we will see, it's quite easy to setup a quick SFT run.

Another good option for SFT is [Hugging Face's TRL](https://huggingface.co/docs/trl), with its [SFT Trainer](https://huggingface.co/docs/trl/en/sft_trainer).

For training, you'll need a machine with a GPU.
You can easily spin up one on Prime Intellect ([documentation](https://docs.primeintellect.ai/quickstart)).

I used a NVIDIA RTX Pro 6000 96GB, but you can use smaller GPUs, with lower `batch_size` values. In any case, this
SFT warm-up will require just a few minutes, so GPU cost is negligible.

Once we have our machine, we can install PRIME-RL by running
```bash
curl -sSL https://raw.githubusercontent.com/PrimeIntellect-ai/prime-rl/main/scripts/install.sh | bash
cd prime-rl
```

If you need Wandb support (for logging training runs) and Hugging Face support (for saving the model), you can run:

```bash
uv run wandb login
uv run hf auth login
```

It's now time to create a configuration file for PRIME-RL. Let's take a look at it (also available [here](../training_configs/primerl_sft.toml)).

```toml
max_steps = 30

[ckpt] # Checkpoint at the end of training

[model]
name = "LiquidAI/LFM2-2.6B"

[data]
name = "anakin87/tictactoe-filtered"
seq_len = 700
batch_size = 32

[optim]
lr = 1e-5
```

Should be quite intuitive.

`seq_len` respect what we saw in the last chapter and makes sure that most of our examples will not be truncated.

We are training the model for 30 steps, with a batch size of 32. It will be
trained on a total of 960 examples; since our cleaned dataset contains 174 examples, the model will see this dataset
~5.5 times. There might be some risk of memorization but we don't care much, since we expect the model here to just
learn formatting; actual gameplay will be learned during RL, which will reshape our model if needed.

Once the configuration file is created (`primerl_sft.toml`), we can launch a tmux session to control our training:
```bash
bash scripts/tmux.sh
```

Then, in the "Trainer" pane, we can start training by running
```
uv run sft @ primerl_sft.toml
```

At the end of the training (<5 minutes using the NVIDIA RTX Pro 6000), we can push the model to Hugging Face:
```bash
uv run hf upload anakin87/LFM2-2.6B-ttt-sft outputs/weights/step_30
```

Final model is avaible [here](https://huggingface.co/anakin87/LFM2-2.6B-ttt-sft).

For more details about PRIME-RL, I recommend their [examples](https://github.com/PrimeIntellect-ai/prime-rl/tree/main/examples).
