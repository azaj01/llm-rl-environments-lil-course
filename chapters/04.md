# Evaluating existing models

In the [previous chapter](03.md), we implemented a Reinforcement Learning environment for Tic Tac Toe.

Now we can use our environment to see how well different language models actually play.

We'll focus on two models:
- `openai/gpt-5-mini`, a good and relatively small closed model.
- [`LiquidAI/LFM2-2.6B`](https://huggingface.co/LiquidAI/LFM2-2.6B), a small, fast open model.
  While it was likely exposed to some reasoning traces during training, we can consider it an instruct
  model: able to follow instructions, but not heavily tuned for thinking.

OpenAI doesn't share parameter counts, but it's safe to assume that `gpt-5-mini` is significantly larger and more capable than `LFM2-2.6B`.

We choose `LiquidAI/LFM2-2.6B` for this step because we will later train this model, but the code
for evaluation is valid for any text generation model available on Hugging Face that is compatible
with the vLLM serving engine.

## `openai/gpt-5-mini`

You can skip any of the following setup steps if you already have the required tools installed.


1. Install uv
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

2. Create and activate a virtual environment
```bash
uv venv
source .venv/bin/activate
```

3. Install Prime CLI
```bash
uv tool install prime && uv tool update-shell
```

4. Install the tictactoe environment
```bash
prime env install anakin87/tictactoe
```

We can now run evaluation using the `prime eval` command. 

Evaluation will use the Prime Intellect
infrastructure by default. If you want to use your own OpenAI API key directly, check out the [Verifiers docs](https://docs.primeintellect.ai/verifiers/evaluation).


### Against a random opponent

Let's first evaluate 100 different matches against a random opponent.

```bash
prime eval run tictactoe -n 100 -r 1 -m openai/gpt-5-mini -a '{"min_random_move_prob": 1.0, "max_random_move_prob": 1.0}'
```

Our output should look something like this:
```
Rewards:
reward: avg - 1.145, std - 0.172
win_reward_func: avg - 0.945, std - 0.172
format_reward_func: avg - 1.000, std - 0.000
invalid_move_penalty_func: avg - 0.000, std - 0.000
num_turns: avg - 3.480, std - 0.519

Info:
is_truncated: avg - 0.000, std - 0.000
stop_conditions: has_final_env_response: 1.000
```

It made zero invalid moves and followed the format perfectly.
From the per-sample metrics, we see 90 Wins, 9 Draws and 1 Losses.

### Against an optimal opponent

Now, let's evaluate the same model against the optimal opponent:
```bash
prime eval run tictactoe -n 100 -r 1 -m openai/gpt-5-mini -a '{"min_random_move_prob": 0.0, "max_random_move_prob": 0.0}'
```

Output:
```
Rewards:
reward: avg - 0.580, std - 0.214
win_reward_func: avg - 0.380, std - 0.214
format_reward_func: avg - 1.000, std - 0.000
invalid_move_penalty_func: avg - 0.000, std - 0.000
num_turns: avg - 4.130, std - 0.844

Info:
is_truncated: avg - 0.000, std - 0.000
stop_conditions: has_final_env_response: 1.000
```

Again, 0 invalid moves, and the format is always respected.
Remembering that it's impossible to beat the optimal opponent, we get 76 Draws and 24 Losses. 

Allowing for some statistical variability, `gpt-5-mini` is excellent at following format instruction and is a good Tic Tac Toe player, though not perfect.

## `LiquidAI/LFM2-2.6B`

We'll serve this small model using vLLM. For this step, you'll need a machine with a GPU.
You can easily spin one up on [Prime Intellect](https://app.primeintellect.ai/dashboard/on-demand-gpus).

I used an NVIDIA RTX Pro 6000 96GB, but you can use way smaller GPUs.


First, repeat the 1-4 steps from the previous section.

5. Install vLLM
```bash
uv pip install vllm
```

6. Launch a vLLM OpenAI-compatible server
```bash
vllm serve LiquidAI/LFM2-2.6B
```

We can now run evaluation using the `prime eval` command from Verifiers.
We point `--api-base-url` to our local vLLM server and set `--api-key-var` to a placeholder key.


### Against a random opponent

Let's first evaluate 100 different matches against a random opponent

```bash
prime eval run tictactoe -n 100 -r 1 -m LiquidAI/LFM2-2.6B --api-base-url http://localhost:8000/v1 --api-key-var PLACEHOLDER_API_KEY -a '{"min_random_move_prob": 1.0, "max_random_move_prob": 1.0}'
```

Our output should contain similar information:
```
Rewards:
reward: avg - 0.471, std - 0.486
win_reward_func: avg - 0.455, std - 0.470
format_reward_func: avg - 0.278, std - 0.323
invalid_move_penalty_func: avg - -0.040, std - 0.049
num_turns: avg - 4.640, std - 1.553

Info:
is_truncated: avg - 0.000, std - 0.000
stop_conditions: has_final_env_response: 0.880, max_turns_reached: 0.120

Timing:
generation: min - 2s, mean - 1m 5s, max - 3m 21s
...
```

The output format is respected less than 30% of the times and there are 40 games with invalid moves.
12% of the times the game ends by reaching the maximum number of turns, meaning that the model tried
several invalid moves in the same game.

Per-sample metrics show 40 Wins, 11 Draws and 49 Losses.

Interesting to notice that generation took about 1m 5s per sample. When we train the model,
we'll check if the trained model becomes faster or slower (a proxy for the number of tokens generated).

### Against an optimal opponent

Now, let's evaluate the same model against the optimal opponent:
```bash
prime eval run tictactoe -n 100 -r 1 -m LiquidAI/LFM2-2.6B --api-base-url http://localhost:8000/v1 --api-key-var PLACEHOLDER_API_KEY -a '{"min_random_move_prob": 0.0, "max_random_move_prob": 0.0}'
```

Output:
```
reward: avg - 0.061, std - 0.184
win_reward_func: avg - 0.055, std - 0.156
format_reward_func: avg - 0.247, std - 0.317
invalid_move_penalty_func: avg - -0.043, std - 0.050
num_turns: avg - 4.030, std - 1.819

Info:
is_truncated: avg - 0.000, std - 0.000
stop_conditions: has_final_env_response: 0.880, max_turns_reached: 0.120

Timing:
generation: min - 4s, mean - 41s, max - 3m 13s
...
```

Again, the output format is respected less than 30% of the times and there are 43 games with invalid moves.
12% of the times the game ends by reaching the maximum number of turns, meaning that the model tried
several invalid moves in the same game.

Per-sample metrics show 11 Draws and 89 Losses.


`LiquidAI/LFM2-2.6B` struggles to follow format instructions and to make valid moves.
It's a weak Tic Tac Toe player, occasionally winning against a random opponent but rarely surviving against an optimal one.

One thing to keep in mind: when looking at wins, draws, and losses, the model's score is a bit inflated because we're forgiving its mistakes, letting it retry moves, and not limiting the number of tokens it produces.
