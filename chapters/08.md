# Reinforcement Learning pt.2: towards Tic Tac Toe mastery

In the [previous chapter](07.md), we used Reinforcement Learning to transform our small model into a competent Tic Tac Toe player.
It dominates random opponents and draws 85% of the time against an optimal opponent.

While impressive, it's not perfect and sometimes falls into fork traps.

In this chapter, we'll try to push our model from "competent" to **Master** status using a second round of RL.

*This chapter is more experimental than the others, pursuing Tic Tac Toe perfection. Read it and have fun!*

## Experimental choices

From the training perspective, we'll perform another Reinforcement Learning run, very similar to [chapter 7](07.md).

But how can we make the model avoid optimal opponent's traps?

**1. Increasing opponent skill**

We previously trained the model against opponents with a probability of making a random move ranging from 20% to 70%, 
so there were no "killer" opponents.

This time, we increase opponent skill: we set the probability of a random move to range from 0% (optimal) to 25%.

Since there is still some noise due to skill diversity, we might be tempted to make our model play against perfect opponents
only.

I learned this the hard way. In a previous experiment with only optimal Minimax players,
the model became overly defensive. When tested against random opponents, it preferred to block their moves instead of
exploiting their errors and winning. We don't want a model that only plays well against perfect players; we want a master that can punish mistakes too.

**2. Encouraging exploration**

Some suboptimal behaviors were strongly reinforced in the previous run, where the
strongest players were not perfect. This led me in several experiments to see this model failing to improve and
"forgetting" the suboptimal strategies. We want the model to explore new strategies.

Then I reflected on *temperature* on Language Model's sampling.

Temperature is a number that rescales the model's logits before the softmax, controlling the entropy of the output distribution:
- `T < 1`: lower entropy; probability of more likely tokens is amplified.
- `T > 1`: higher entropy; less likely tokens have increased probability of getting selected.

For an intuitive intro to temperature in Language Models, check out [this blog post](https://blog.lukesalamone.com/posts/what-is-temperature/).

To stimulate exploration, I tried increasing the temperature to 1.25.

This is risky, though: if the temperature is too high, the model can start generating gibberish. We need to monitor this run closely.

Based on these choices, here are the key parts of our new training configuration (full file [here](../training_configs/vfrltrainer_rl2.toml)).

```toml
model = "anakin87/LFM2-2.6B-ttt-rl-merged"

[env]
id = "anakin87/tictactoe"

[env.args]
min_random_move_prob = 0.0
max_random_move_prob = 0.25
max_turns = 7
num_examples = 24000
num_groups = 60

[inference]
...

[trainer]
gpus = 1

[trainer.args]
run_name = "tictactoe"

use_liger = false

micro_batch_size = 24
rollouts_per_example = 8
batch_size = 480

lora_rank = 8

max_tokens = 64
max_seq_len = 384

max_steps = 400

learning_rate = 3e-5
temperature = 1.25

...
```

Some other changes with respect to the previous configuration:
- This time I wanted to iterate fast, so I used 2 NVIDIA H200 (141 GB VRAM). I updated `micro_batch_size` and `batch_size` to fully utilize their memory. A larger batch size also improves training stability.
- I reduced `learning_rate` from `5e-5` to `3e-5`. The model is already good; we don't want to break it, just refine it.
- `max_tokens` and `max_seq_len` are set to lower values than previously. In the previous run, the maximum completion tokens stayed around 40, and we don't expect them to grow much.


## Training

We are ready to train. On a machine with two GPUs, let's setup the required packages (see [previous chapter](07.md) for details).


## Merge the LoRA adapter

To evaluate our model, let's merge the base model with our new LoRA adapter ([`anakin87/LFM2-2.6B-ttt-rl-2`](https://huggingface.co/anakin87/LFM2-2.6B-ttt-rl-2)):

```python
# ! pip install peft transformers torch

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

BASE_MODEL_ID = "anakin87/LFM2-2.6B-ttt-rl-merged"
LORA_ADAPTER_ID = "anakin87/LFM2-2.6B-ttt-rl-2"
MERGED_MODEL_ID = "anakin87/LFM2-2.6B-mr-tictactoe"

tokenizer = AutoTokenizer.from_pretrained(BASE_MODEL_ID)

base_model = AutoModelForCausalLM.from_pretrained(
    BASE_MODEL_ID,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

model = PeftModel.from_pretrained(base_model, LORA_ADAPTER_ID)

model = model.merge_and_unload()

model.eval()

model.push_to_hub(MERGED_MODEL_ID, safe_serialization=True)
tokenizer.push_to_hub(MERGED_MODEL_ID)
```

The merged standalone model is available as [`anakin87/LFM2-2.6B-mr-tictactoe`](anakin87/LFM2-2.6B-mr-tictactoe).

I chose a new name for this finalized model. Interpret as you like: Mister Tic Tac Toe, Master Tic Tac Toe, ... 😊❌⭕

## Conclusion and next up

We started this course with a small, open model that struggled to format moves and lost 89% of the time against an optimal Tic Tac Toe opponent.

With the help of a verifiable environment, we:
- Standardized the model's output using Supervised Fine-Tuning on synthetic data.
- Trained it with Reinforcement Learning against opponents with varying skill.
- Refined it, encouraging exploration in the same environment.

The result is a 2.6B parameter model that plays nearly perfect Tic Tac Toe, outperforming gpt-5-mini on this specific task.

This teaches us a lesson about AI agents: general-purpose models are powerful, but **small, specialized models trained with RL in verifiable environments can beat them on specific tasks, at a fraction of the cost**.
